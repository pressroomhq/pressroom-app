# Blog Post

Write a technical blog post that's useful enough to bookmark. SEO-aware but written for humans first.

## Format

- 800-1500 words
- Markdown with H2 subheadings every 200-300 words
- Code snippets in fenced blocks with language tags where relevant
- Include specific examples, benchmarks, or comparisons where possible

## Structure

- Title should include the primary keyword naturally — no keyword stuffing, but be specific about what this post covers
- No fluff intro paragraphs. Start with the point. The reader clicked because the title promised something — deliver immediately.
- Each H2 section should advance the argument or teach something new. If a section doesn't add value, cut it.
- Include [LINK: topic] markers where internal links to related content would be useful
- End with what's next — the natural follow-up question, a related technique to explore, or what you'd do differently. Not a generic "In conclusion" paragraph.

## Style

- Technical depth appropriate to the audience — don't over-explain basics, don't hand-wave complex parts
- Code snippets should be real and runnable, not pseudocode (unless the concept genuinely calls for pseudocode)
- Opinions are fine — "We chose X over Y because Z" is more useful than neutrally listing options
- Write in second person ("you") or first person plural ("we") — not third person passive

## What to avoid

- "In today's fast-paced world" or any variation of that opener
- Defining common terms your audience already knows
- Stock photo energy in the writing (vague, generic, interchangeable with any other post)
- Promising things the post doesn't deliver
