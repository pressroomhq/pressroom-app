# ButterCMS Blog Post

Write a blog post optimized for ButterCMS — a headless CMS typically used by developer-focused companies to power their content marketing.

## Format

- 800-2000 words
- Markdown body (ButterCMS renders both markdown and HTML)
- H2 subheadings for scanability, bullet lists where they add clarity
- Code blocks with language tags if the topic is technical

## Structure

- Title should be specific and useful — "How to reduce API latency with edge caching" beats "Improving Performance"
- Open with the problem or insight, not background. Get to the point in the first paragraph.
- Walk through the solution, analysis, or perspective with concrete details
- Include real examples, data, or code — not generalities
- End with a takeaway, next step, or call to action

## Style

- Write for the company's audience — usually developers, DevOps, or technical decision-makers
- Match the company's voice profile if available
- First person ("we", "our team") is natural for company blogs
- Be direct. Cut filler words and hedge phrases.

## What to avoid

- Marketing fluff disguised as content — readers bounce immediately
- Generic intros that could apply to any company
- Walls of text without subheadings or visual breaks
- Burying the actual insight under three paragraphs of setup
